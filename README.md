**# Proyecto_Intermodular_Grupo3**# **Objetivos del Proyecto — TodoBit**

En este apartado se describen los problemas que TodoBit pretende resolver y los objetivos establecidos para el desarrollo de la plataforma. Cada objetivo sigue la metodología SMART (Específico, Medible, Alcanzable, Relevante y Temporal) para garantizar su viabilidad y seguimiento.

Problema que se pretende resolver:

- Las plataformas de juegos gratuitas existentes (FRIV, AGAME) no ofrecen sistemas de perfil de usuario ni guardado de progreso.

- Ningún competidor directo combina el estilo retro con funciones modernas como registro, logros y leaderboard.

- No existe control de contenido para menores en la mayoría de plataformas del sector.

## **Objetivo 1 — Plataforma de juegos retro funcional**

Desarrollar y publicar una página web operativa con al menos 2 juegos de estilo retro antes de la fecha de entrega del proyecto.

| **S — Específico** | Crear una web con al menos 2 juegos retro jugables desde el navegador sin necesidad de descarga. |
| --- | --- |
| **M — Medible** | La web estará considerada completada cuando los 2 juegos funcionen correctamente en los principales navegadores (Chrome, Firefox, Edge). |
| **A — Alcanzable** | El equipo de 4 personas (Bradan, Artur, Jose y Etienne) cuenta con los conocimientos de 1º DAW para desarrollar los juegos con tecnologías HTML5/JS. |
| **R — Relevante** | Los juegos retro representan un nicho diferenciador frente a competidores como FRIV o CrazyGames, aportando identidad propia a TodoBit. |
| **T — Temporal** | Completado antes de la fecha límite del curso de 1º DAW. |

## **Objetivo 2 — Sistema de registro y login de usuarios**

Implementar un sistema de registro e inicio de sesión que permita a los usuarios guardar su progreso y puntuaciones en un plazo máximo de 3 semanas desde el inicio del desarrollo.

| **S — Específico** | Crear formularios de registro y login funcionales con almacenamiento de datos de usuario (nombre, edad, logros y partidas guardadas). |
| --- | --- |
| **M — Medible** | El sistema estará completo cuando un usuario pueda: registrarse, iniciar sesión, guardar una partida y visualizar sus logros correctamente. |
| **A — Alcanzable** | Basado en el modelo de CrazyGames (competidor top 2), cuya implementación de login es referencia directa para el equipo. |
| **R — Relevante** | El login es un diferenciador clave frente a FRIV (sin registro) y añade fidelización de usuarios a la plataforma. |
| **T — Temporal** | Funcional en las 3 primeras semanas de desarrollo del proyecto. |

## **Objetivo 3 — Control de edad y contenido family friendly**

Incorporar un sistema de verificación de edad que adapte el contenido y los efectos de los juegos para menores de edad desde el primer lanzamiento de la web.

| **S — Específico** | Al registrarse, el usuario deberá introducir su fecha de nacimiento. Los menores de edad verán una versión del juego con efectos reducidos y contenido adaptado. |
| --- | --- |
| **M — Medible** | El sistema estará validado cuando se compruebe que usuarios registrados como menores reciben correctamente la versión restringida y los adultos la versión completa. |
| **A — Alcanzable** | La verificación de edad se puede implementar con lógica condicional sobre el campo de fecha de nacimiento registrado. |
| **R — Relevante** | Garantiza que la plataforma sea segura para todos los públicos, especialmente para el rango de 7–16 años identificado como audiencia principal del sector (ver análisis de AGAME). |
| **T — Temporal** | Integrado en el sistema de registro desde el lanzamiento inicial de la web. |

## **Objetivo 4 — Leaderboard con mejores puntuaciones**

Diseñar e implementar un marcador de puntuaciones globales (leaderboard) visible para todos los usuarios registrados antes del lanzamiento de la web.

| **S — Específico** | Crear una tabla de clasificación que muestre las 10 mejores puntuaciones de cada juego, con nombre de usuario y puntuación. |
| --- | --- |
| **M — Medible** | El leaderboard estará completo cuando se actualice automáticamente tras cada partida y muestre los datos ordenados correctamente. |
| **A — Alcanzable** | Su implementación se apoya en la base de datos de usuarios ya creada en el Objetivo 2. |
| **R — Relevante** | Fomenta la competitividad y la retención de usuarios, aumentando el tiempo de uso de la plataforma y diferenciándola de competidores sin esta función. |
| **T — Temporal** | Disponible desde el lanzamiento público de la web. |
